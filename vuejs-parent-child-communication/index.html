VueJs Parent Child Communication Vue.js 父子组件通信

We’ve been learning about how components and specifically single file components work in VueJS. In this tutorial, we’ll expand our learning about components in Vue by looking at how communication happens between components. Components in and of themselves are great as they set up reusable pieces of code where we can use all of the features VueJS provides. Now we want to build connections between those components so that actions in one component can update a different component in the application. We’ll focus on parent to child and child to parent communication in this tutorial.

我们一直在学习 VueJS 中的组件 (尤其是单个文件组件) 如何工作. 在本教程中, 我们将通过研究组件之间的通信方式来扩展对 Vue 组件的了解. 组件本身本身很棒, 因为它们设置了可重用的代码段, 我们可以在其中使用 VueJS 提供的所有功能. 现在我们要在这些组件之间建立连接, 以便一个组件中的动作可以更新应用程序中的另一个组件. 在本教程中, 我们将重点介绍父母与子女以及子女与父母之间的交流.

Parent To Child Communication In Vue

To move data from a parent component to a child component in Vue we use something called props. ReactJS also uses a similar convention for sharing data. Props is short for “properties” and is referring to properties set from outside, such as from the parent component. In order to tell the vue child component that it will receive data from outside of it’s own instance, you need to set up the props property in the Vue object of the child component. This property holds an array of strings, with each string representing a property which can be set from the parent. Note that props are strictly for one way communication from the parent to the child, and you do not want to try and change the value of a prop directly in the child component. Otherwise, you will get an error of something like “Avoid mutating a prop directly since the value will be overwritten whenever the parent component re-renders. Instead, use a data or computed property based on the prop’s value.”

要将数据从 Vue 中的父组件移动到子组件, 我们使用了称为 props 的东西. ReactJS 也使用类似的约定来共享数据. 道具是 “属性” 的缩写, 是指从外部 (例如, 从父级组件) 设置的属性. 为了告诉 vue 子组件它将从自己实例的外部接收数据, 您需要在子组件的 Vue 对象中设置 props 属性. 此属性包含一个字符串数组, 每个字符串代表一个可以从父级设置的属性. 请注意, 道具严格来说是从父母到孩子的一种单向通信, 并且您不想尝试直接在子组件中更改道具的值. 否则, 会出现类似 “避免直接更改道具的错误”, 因为每当父组件重新渲染时, 该值就会被覆盖. 而是根据道具的值使用数据或计算属性.”

Parent Component Uses An Attribute Binding

In order to pass the data down from the Parent Component to the Child Component, we can now visit the parent component and set up an attribute binding which uses the same name as the prop from the child component. Note that we are inside the parent component, but we are rendering a child component using it’s custom tag of . It is on this tag where we set up the bound attribute. Now, since we are using parentmessage as the attribute name, then in the Child we will need props: ['parentmessage'] as a prop. In our Parent we pass the data down using <child-card :parentmessage="parentmessage"></child-card>.

为了将数据从父组件传递到子组件, 我们现在可以访问父组件并设置属性绑定, 该属性绑定使用与子组件中的 prop 相同的名称. 请注意, 我们位于父组件内部, 但是我们正在使用其 <child-card> 的自定义标记来渲染子组件. 我们在此标记上设置 bound 属性. 现在, 由于我们使用 parentmessage 作为属性名称, 因此在 Child 中, 我们需要 props: ['parentmessage'] 作为 prop. 在父级中, 我们使用 <child-card:parentmessage ="parentmessage"> </ child-card> 向下传递数据.

ParentCard.vue

<template>
  <div>
    <div class="card m-2" style="width: 40rem;">
      <div class="card-body">
        <h5 class="card-title" v-text="thecardtitle"></h5>
        <button @click="sendMessage" class="btn btn-info">
          Send Child A Message
        </button>
        <child-card :parentmessage="parentmessage"></child-card>
      </div>
    </div>
  </div>
</template>

<script>
  import ChildCard from './ChildCard.vue';
  export default {
    components: {ChildCard},
    data() {
      return {
        thecardtitle: 'Parent Component!',
        parentmessage: '',
      };
    },
    methods: {
      sendMessage() {
        this.parentmessage = '<b>Message From Parent:</b> Do Your Homework';
      },
    },
  };
</script>

Child Component uses props Object

So in this snippet below we have a Child component which has a prop setup and a string value of parentmessage in it’s array. What this indicates is that parentmessage can be set from the outside, or from the parent component. That is exactly what we are doing in the section above. The string name given for the prop, in our case parentmessage must match the property name used in the template section of this component.

因此, 在下面的代码段中, 我们有一个 Child 组件, 该组件具有道具设置和数组中的 parentmessage 字符串值. 这表明可以从外部或从父组件设置父消息. 这正是我们在上一节中所做的. 为道具指定的字符串名称 (在我们的示例中为 parentmessage) 必须与该组件的模板部分中使用的属性名称匹配.

ChildCard.vue

<template>
  <div>
    <div class="card m-3" style="width: 25rem;">
      <div class="card-body">
        <h5 class="card-title" v-text="thecardtitle"></h5>
        <p class="card-text" v-html="thecardbody"></p>
        <div
          v-if="parentmessage"
          class="card-text alert alert-warning"
          v-html="parentmessage"
        ></div>
      </div>
    </div>
  </div>
</template>

<script>
  export default {
    props: ['parentmessage'],
    data() {
      return {
        thecardtitle: 'Child Component!',
        thecardbody: "I'm just a child.",
      };
    },
  };
</script>

Parent To Child Communication Prop Demo

So let’s see this little bit of code in action. Below we have the rendered ParentCard.vue parent component, and a nested ChildCard.vue child component. In the child, we are using a v-if to conditionally display an alert with a message from the parent. If there is no message, then we do not display the alert. So when the page first renders, the initial value of parentmessage is just an empty string. We can see this in the data() method of the Parent Component. That is why at first, the Child component does not display any alert message. However, if the user clicks the “Send Child A Message” button in the Parent component, then a sendMessage() function triggers. This sets the parentmessage variable to “<b>Message From Parent:</b> Do Your Homework“. Since this variable is bound to the child using :parentmessage="parentmessage", and the child component is accepting that value via props: [“parentmessage”], then the child component updates with the alert message of “Message From Parent: Do Your Homework”. Pretty cool!

因此, 让我们看一下这些实际的代码. 在下面, 我们提供了呈现的 ParentCard.vue 父组件和一个嵌套的 ChildCard.vue 子组件. 在孩子中, 我们使用 v-if 来有条件地显示警报以及来自父母的消息. 如果没有消息, 那么我们不会显示警报. 因此, 当页面首次呈现时, parentmessage 的初始值只是一个空字符串. 我们可以在父组件的 data() 方法中看到这一点. 这就是为什么起初, 子组件不显示任何警报消息的原因. 但是, 如果用户单击 “父” 组件中的 “向子节点发送消息” 按钮, 则会触发 sendMessage() 函数. 这会将 parentmessage 变量设置为 “<b>Message From Parent:</b> Do Your Homework“. 由于此变量是使用: parentmessage =”parentmessage” 绑定到子级的, 并且子级组件正在通过 props [[“parentmessage”] 接受该值, 因此该子级组件将更新为 “来自父母的消息: 您的信息家庭作业”. 太酷了！

Child To Parent Communication In Vue

What about going in the other direction? How can we communicate from a Child component to a Parent Component in VueJS? For this we can emit a custom event in the Child Component, and listen for that emitted event in the Parent Component. Let’s add a little bit of markup to our existing demo code to create a custom event.

朝另一个方向去怎么办？我们如何在 VueJS 中从子组件与父组件进行通信？为此, 我们可以在子组件中发出一个自定义事件, 并在父组件中侦听该发出的事件. 让我们在现有的演示代码中添加一些标记以创建自定义事件.

The Child Emits a Custom Event

First off in the <template> section of the child component, we will add a click handler of @click=”ok” like so.

首先, 在子组件的 <template> 部分中, 我们将添加 @ click =”ok” 的点击处理程序, 如下所示.

ChildCard.vue

<template>
  <div>
    <div class="card m-3" style="width: 25rem;">
      <div class="card-body">
        <h5 class="card-title" v-text="thecardtitle"></h5>
        <p class="card-text" v-html="thecardbody"></p>
        <div
          v-if="parentmessage"
          class="card-text alert alert-warning"
          v-html="parentmessage"
        ></div>
        <button v-if="parentmessage" @click="ok" class="btn btn-success">
          Ok
        </button>
      </div>
    </div>
  </div>
</template>

What this says is that when we click the Ok button, we want to run a method named ok(). Let’s set up that method to emit a custom event when it triggers. We pass a string of ‘finished’ to the $emit function. We can choose any name we like, but this makes sense in our case. We want to send a message from the child back to the parent that we are finished.

这就是说, 当我们单击 “确定” 按钮时, 我们要运行一个名为 ok() 的方法. 让我们将该方法设置为在触发时发出自定义事件. 我们将一串 “完成” 传递给 $emit 函数. 我们可以选择任何喜欢的名称, 但这在我们的情况下是有道理的. 我们希望将信息从孩子发回给父母, 我们已经完成了.

ChildCard.vue

<script>
  export default {
    props: ['parentmessage'],
    data() {
      return {
        thecardtitle: 'Child Component!',
        thecardbody: "I'm just a child.",
      };
    },
    methods: {
      ok() {
        this.$emit('finished');
      },
    },
  };
</script>

The Parent Listens For The Custom Event

Now we can move back up to the Parent component and where we make use of the <child-card> custom tag, we can now attach an event listener for our custom event using @finished=”finished”. This means we are going to want to run a finished() function inside the Parent component. Both the custom listener and the function it triggers are highlighted here.

现在, 我们可以返回到 Parent 组件, 并在其中使用 <child-card> 自定义标记, 现在可以使用 @ finished =”finished” 为我们的自定义事件附加一个事件侦听器. 这意味着我们将要在 Parent 组件内运行 finish() 函数. 自定义侦听器及其触发的功能都在此处突出显示.

ParentCard.vue

<template>
  <div>
    <div class="card m-2" style="width: 40rem;">
      <div class="card-body">
        <h5 class="card-title" v-text="thecardtitle"></h5>
        <button @click="sendMessage" class="btn btn-info">
          Send Child A Message
        </button>
        <child-card
          :parentmessage="parentmessage"
          @finished="finished"
        ></child-card>
      </div>
    </div>
  </div>
</template>

<script>
  import ChildCard from './ChildCard.vue';
  export default {
    components: {ChildCard},
    data() {
      return {
        thecardtitle: 'Parent Component!',
        parentmessage: '',
      };
    },
    methods: {
      sendMessage() {
        this.parentmessage = '<b>Message From Parent:</b> Do Your Homework';
      },
      finished() {
        this.parentmessage = '';
      },
    },
  };
</script>

Child To Parent Communication Emit Event Demo

In this demo you first click the “Send Child A Message” button. That sends the message down to the child component, and it renders the message along with a new button. Now, we can click the “Ok” button, and it emits that custom ‘finished’ event. In the Parent, we were listening for that custom event and on hearing it we trigger the finished() function to set the parentmessage back to an empty string. Finally, everything re renders and we are back to the starting point. Now we have the Parent sending messages to the Child and the Child responding as well as the Child sending messages to the Parent and the Parent responding. Cool!

在此演示中, 您首先单击 “向孩子发送消息” 按钮. 它将消息向下发送到子组件, 并将消息和新按钮一起呈现. 现在, 我们可以单击 “确定” 按钮, 它将发出自定义的 “完成” 事件. 在父级中, 我们正在监听该自定义事件, 并在听到该事件后, 我们触发 finish() 函数将父级消息设置回空字符串. 最后, 一切都重新渲染, 我们回到了起点. 现在, 我们让父级向子级和父级发送消息, 以及子级向父级和父级发送消息. 凉！

Child To Parent Communication via Callback Function

There is another way to send a message back to the parent from the child if you don’t want to emit a custom event. What you could do is instead of defining the ok() method on the child, you can define it on the parent. Once it is defined on the parent, you can pass another prop from the parent to the child. So what we have here in the ParentCard.vue file is the ok() method defined and highlighted, and the binding on the <child-card> defined and highlighted.

如果您不想发出自定义事件, 则可以通过另一种方式将消息从孩子发送回父母. 您可以做的是代替在子级上定义 ok() 方法, 而可以在父级上定义它. 在父项上定义后, 您可以将另一个道具从父项传递给子项. 因此, 在 ParentCard.vue 文件中, 我们定义并突出显示了 ok() 方法, 并定义并突出显示了 <child-card> 上的绑定.

ParentCard.vue

<template>
  <div>
    <div class="card m-2" style="width: 40rem;">
      <div class="card-body">
        <h5 class="card-title" v-text="thecardtitle"></h5>
        <button @click="sendMessage" class="btn btn-info">
          Send Child A Message
        </button>
        <child-card
          :parentmessage="parentmessage"
          :ok="ok"
          @finished="finished"
        ></child-card>
      </div>
    </div>
  </div>
</template>

<script>
  import ChildCard from './ChildCard.vue';
  export default {
    components: {ChildCard},
    data() {
      return {
        thecardtitle: 'Parent Component!',
        parentmessage: '',
      };
    },
    methods: {
      sendMessage() {
        this.parentmessage = '<b>Message From Parent:</b> Do Your Homework';
      },
      finished() {
        this.parentmessage = '';
      },
      ok() {
        this.finished();
      },
    },
  };
</script>

Now we need to update our props on the child component. What this does is enable the passing of a callback function from the parent to the child via props. We can set that up like so.

现在我们需要在子组件上更新道具. 这样做是通过道具将回调函数从父级传递到子级. 我们可以这样设置.

ChildCard.vue

<template>
  <div>
    <div class="card m-3" style="width: 25rem;">
      <div class="card-body">
        <h5 class="card-title" v-text="thecardtitle"></h5>
        <p class="card-text" v-html="thecardbody"></p>
        <div
          v-if="parentmessage"
          class="card-text alert alert-warning"
          v-html="parentmessage"
        ></div>
        <button v-if="parentmessage" @click="ok" class="btn btn-success">
          Ok
        </button>
      </div>
    </div>
  </div>
</template>

<script>
  export default {
    props: ['parentmessage', 'ok'],
    data() {
      return {
        thecardtitle: 'Child Component!',

        thecardbody: "I'm just a child.",
      };
    },
  };
</script>

<style></style>

The interaction between the parent and child is the same as it was before when we were emitting a custom event. You can use whichever option seems easier to you, but it does seem that emitting a custom event is the more popular of the two ways to communicate from a child component to a parent component.

父子之间的交互与发出自定义事件之前的交互相同. 您可以使用对您来说似乎更容易的任何一个选项, 但似乎发出自定义事件是从子组件到父组件进行通信的两种方式中比较流行的一种.
